# TeleClaw 公众号草稿执行指令

## 1. 执行目标

TeleClaw 的唯一目标是：

在医生审核通过后，打开微信公众号后台，把医生确认过的文章内容保存为草稿。

TeleClaw 不允许发布、发表或群发文章。

## 2. 执行前必须确认

开始前必须确认以下事项全部满足：

- [ ] 医生已经填写脱敏科普卡
- [ ] 星辰智能体已经生成公众号文章草稿
- [ ] 星辰智能体已经生成风险审核报告
- [ ] 医生已经阅读并审核文章
- [ ] 医生审核结论为“通过”
- [ ] 文章结尾包含固定声明
- [ ] TeleClaw 任务包明确只要求保存草稿

如果任意一项没有满足，TeleClaw 必须停止执行。

## 3. 固定禁止事项

TeleClaw 严禁执行以下动作：

- 点击发布
- 点击发表
- 点击群发
- 点击确认发布
- 点击任何会让文章对外可见的按钮
- 绕过医生审核
- 修改医生已经审核通过的专业内容
- 添加未经医生确认的新内容
- 处理原始心理咨询记录
- 处理真实患者或来访者身份信息

## 4. 可复制的 TeleClaw 执行指令

将下面内容与医生审核通过后的任务包一起交给 TeleClaw。

```markdown
# TeleClaw 执行指令：保存微信公众号草稿

你现在要帮助医生在本地电脑上创建微信公众号草稿。

你的任务只包括：
1. 打开微信公众号后台。
2. 进入图文草稿或新建图文页面。
3. 按任务包填写标题、正文、摘要等内容。
4. 保存为草稿。
5. 停止在草稿保存完成的状态。

你绝对不允许：
1. 点击发布。
2. 点击发表。
3. 点击群发。
4. 点击确认发布。
5. 点击任何会让文章公开的按钮。
6. 在医生没有审核通过时创建草稿。
7. 添加未经医生确认的新内容。

执行前请检查任务包中的医生审核状态。

如果医生审核状态不是“通过”，请停止执行，并提示：医生未审核通过，不能创建公众号草稿。

如果页面出现发布、发表、群发、确认发布等按钮，请不要点击。

完成标准：
文章已经保存在微信公众号后台草稿箱中，并且没有被发布、发表或群发。
```

## 6. 异常情况处理

### 情况一：医生审核状态不是“通过”

停止执行。

提示：

> 医生未审核通过，不能创建公众号草稿。

### 情况二：任务包缺少固定结尾声明

停止执行。

提示：

> 文章缺少固定声明，请先补充并由医生重新审核。

### 情况三：页面出现发布、发表、群发按钮

不要点击。

只寻找保存草稿相关按钮。如果无法确认哪个按钮是保存草稿，请停止并请医生确认。

### 情况四：正文内容和任务包不一致

停止执行。

提示：

> 当前内容与医生审核通过版本不一致，请医生确认后再继续。

### 情况五：公众号后台要求二次确认

如果二次确认与保存草稿有关，可以继续。

如果二次确认与发布、发表、群发有关，必须取消或停止。

TeleClaw 执行完成后，只需要反馈：

- 草稿是否已经保存
- 是否有任何异常
- 是否没有发布、发表、群发
