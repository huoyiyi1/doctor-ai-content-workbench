# 医生公众号 AI 内容工作台设计规范

## 设计原则

- 页面只展示当前模块相关内容，避免跨模块内容混杂。
- 操作路径优先清晰，不追求装饰感。
- 医生审核、风险提示、发布边界要始终可见且表达克制。
- 所有页面文案使用中文，按钮用动词开头。

## 导航

- 顶部模块导航固定为四个入口：新建文章页、任务列表页、任务详情 / 编辑页、排版发布页。
- 当前模块需要有明确高亮，并显示“当前｜模块名”。
- 模块跳转使用 URL 参数保存页面和任务，例如 `?page=detail&task_id=xxx`。

## 按钮

- 标准按钮高度不低于 48px。
- 标准圆角为 10px。
- 主按钮只用于关键推进动作，例如保存、生成、提交审核、进入排版。
- 次按钮用于局部动作，例如重新生成、恢复、复制、状态标记。
- 同一行按钮视觉规格保持一致，不混用明显不同的尺寸和圆角。
- 禁用按钮必须说明原因，例如“请先点击一键美化排版”。

## 复制按钮

- 复制 Markdown、复制美化 Markdown、排版发布页复制 Markdown 使用统一按钮样式。
- 点击后在按钮旁显示“已复制 Markdown”或“已复制美化 Markdown”。
- 没有美化 Markdown 时，复制美化 Markdown 禁用。
- 复制按钮下方要留出空间，不能压住文本框或预览区域。

## 表单与任务选择

- 任务列表页可以加强当前选中任务展示，帮助用户确认管理对象。
- 任务详情 / 编辑页的任务选择保持简洁，只显示任务编号，避免顶部区域过重。
- 用户手动选择任务时，不允许被 URL 中的旧任务编号强制覆盖。

## 状态与加载

- 调用大模型的按钮必须显示“正在xxx，请稍等...”。
- 完成后显示简短成功提示。
- 失败时显示清楚错误提示，不吞掉错误。

## 内容安全边界

- 不处理原始心理咨询记录。
- 不使用真实来访者故事。
- 不做诊断，不给药物建议，不承诺疗效。
- 未审核通过，不进入排版发布页。
- Raphael 只用于排版预览，最终发布必须人工完成。
