# 公众号内容自动生成工作台

这是第一版可演示产品，用于本地运行。

它帮助专业人士、知识型创作者或运营人员维护脱敏知识库，基于知识库生成公众号选题，再把选题整理成公众号文章，并完成审核、文章编辑、图片提示词生成和 Raphael 排版预览。

第一版不接：

- 飞书 API
- GPT Image API
- 公众号 API
- 自动发布
- 登录系统

## 核心边界

- 不处理原始心理咨询记录。
- 不使用真实来访者故事。
- 不做诊断。
- 不给药物建议。
- 不承诺疗效。
- 不制造焦虑。
- 不输出 `<think>`、`Thinking Process`、`推理过程`。
- 未审核通过，不进入排版发布页。
- 当前版本不接真实图片生成模型 API，但支持图片计划、手动上传图片、填写图片链接和生成图文版 Markdown。
- 知识库只保存脱敏后的医生风格、选题素材、参考资料、合规规则和历史文章，不保存真实来访者身份信息。
- 每周内容计划只生成选题和文章任务，不做自动定时发布。

正文结尾必须包含：

```text
本文由 AI 辅助整理，医生审核后发布。
本文仅作心理健康科普，不替代诊断、治疗或个体化心理咨询。
```

## 如何运行

在项目文件夹中运行：

```bash
pip install -r requirements.txt
streamlit run app.py
```

如果你使用本项目自带的虚拟环境，可以运行：

```bash
.venv/bin/streamlit run app.py
```

启动后打开本地网页：

```text
http://127.0.0.1:8501
```

## 如何部署给别人试用

当前项目已经支持两种运行方式：

- 本地运行：默认使用 `data/tasks.db` SQLite 数据库。
- 云端部署：如果配置了 `DATABASE_URL`，系统会自动使用 Postgres 数据库。

给别人试用时，推荐使用云端部署，不要再依赖你电脑上的 `127.0.0.1`。

### 方案 A：Streamlit Community Cloud，最快演示

适合：先发给少量朋友、同事、医生看演示。

步骤：

1. 把项目上传到 GitHub。
2. 确认不要上传 `.env`、`.streamlit/secrets.toml`、`data/tasks.db`、`outputs/*.md`。
3. 打开 Streamlit Community Cloud，新建 App。
4. 选择 GitHub 仓库。
5. 入口文件填写：

```text
app.py
```

6. 在 Streamlit Cloud 的 Secrets 中填写：

```toml
DEEPSEEK_API_KEY = "你的 DeepSeek API Key"
DEEPSEEK_API_BASE = "https://api.deepseek.com"
DEEPSEEK_MODEL = "deepseek-chat"

# 演示期可以先留空；多人连续试用建议填写外部 Postgres。
DATABASE_URL = ""
```

部署完成后，你会得到一个类似下面的公开地址：

```text
https://你的应用名.streamlit.app
```

注意：如果 `DATABASE_URL` 留空，云端会退回使用本地 SQLite。这个方式适合短期演示，但云端重启后数据可能丢失，不适合作为长期试用数据库。

### 方案 B：Streamlit Cloud + Supabase Postgres，推荐给别人连续试用

适合：你希望别人连续使用，任务数据不要因为云端重启而丢失。

步骤：

1. 创建一个 Supabase 项目。
2. 复制 Supabase 的 Postgres 连接串。
3. 在 Streamlit Cloud Secrets 中填写：

```toml
DEEPSEEK_API_KEY = "你的 DeepSeek API Key"
DEEPSEEK_API_BASE = "https://api.deepseek.com"
DEEPSEEK_MODEL = "deepseek-chat"
DATABASE_URL = "postgresql://用户名:密码@主机:端口/数据库名"
```

应用启动时会自动创建和迁移 `tasks`、`knowledge_items`、`weekly_plans`、`weekly_topics` 表。

### 方案 C：Render，一体化部署

适合：你希望更接近正式产品部署。

项目里已经提供了 `render.yaml`。

步骤：

1. 把项目上传到 GitHub。
2. 在 Render 创建 Web Service。
3. 选择当前仓库。
4. Render 会读取 `render.yaml`。
5. 在 Render Environment Variables 里填写：

```text
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_API_BASE=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
DATABASE_URL=你的 Postgres 连接串
```

Render 部署完成后，会生成一个公开 URL，可以直接发给别人试用。

## 部署前检查清单

- `.env` 没有上传。

- `data/tasks.db` 没有上传。
- `outputs` 里没有真实业务内容。
- `input/tasks.csv` 里没有真实患者或来访者信息。

## 页面结构

当前 V2 重构后的左侧主导航只保留 5 个入口：

- 内容工作台
- 知识库
- 计划生成
- 历史稿件
- 设置

详情页、排版页、新增今日素材页会在流程中打开，不再作为复杂的一级入口展示。

界面设计和交互统一遵循：[医生公众号 AI 内容工作台设计规范](docs/design_guidelines.md)。

### 1. 内容工作台

首页展示三个主入口：

- 直接生成一篇
- 基于知识库生成
- 每周计划生成

顶部提供“新增今日素材”按钮。页面下方展示待审核稿件、已生成未发布数量、本周计划剩余数量，以及最近生成的 5 篇稿件。

### 2. 直接生成一篇

只展示脱敏科普卡表单和“保存文章任务”按钮：

- article_topic：文章主题
- target_reader：目标读者
- doctor_name：医生署名
- safe_scene：脱敏场景
- common_misunderstanding：常见误区
- doctor_viewpoint：医生核心观点
- forbidden_info：禁止出现的信息
- risk_tags：风险标签
- writing_style：文章风格
- article_length：文章长度，支持短文 800 字、中等 1200 字、长文 1800 字、深度 3000 字、最长 4000 字
- image_style：图片风格
- layout_style：排版风格

### 3. 历史稿件

历史稿件是一个稿件库，用来集中管理已经生成过的内容。

支持筛选：

- 全部
- 待审核
- 已通过
- 已发布
- 已废弃

表格会展示：

- 标题
- 来源：直接生成、知识库生成、计划生成
- 使用知识库
- 状态
- 风险等级
- 生成时间
- 更新时间

选中一篇稿件后，可以：

- 查看稿件
- 继续编辑
- 复制 Markdown
- 标记已发布
- 标记废弃

未审核通过的稿件不能复制最终发布内容。要先进入“文章详情 / 审核发布页”完成审核。

### 4. 文章详情 / 审核发布页

可以完成：

- 查看文章标题、状态、风险等级、来源知识库和生成时间。
- 在左侧编辑干净的公众号正文 Markdown，并可切换“编辑模式 / 预览模式”。
- 正文中使用 `{{image:inline_1}}` 这类图片占位符，不再混入图片提示词。
- 预览模式会把已有图片链接的占位符渲染为图片；没有链接时显示灰色插图占位。
- 在右侧查看当前状态、风险等级、风险审核报告、审核建议、引用内容和重复检查结果。
- 在右侧“图片生成中心”查看封面图、正文配图和备用图的计划、生成状态、链接状态和插入状态。
- 保存标题、摘要和正文修改。
- 保存并重新审核，或在退回后提交复审。
- 润色正文。
- 重新生成标题。
- 重新生成摘要。
- 重新生成图片计划，并更新封面图、正文配图和备用图。
- 一键优化排版，先生成预览，确认后才应用。
- 重新风险审核。
- 退回修改，并必须填写修改意见。
- 审核通过。
- 复制文字版 Markdown。
- 复制图文版 Markdown。
- 打开 Raphael 排版预览。
- 标记为已发布。

退回修改支持两种方式：

- AI 根据修改意见重写正文：先生成预览，确认后才应用。
- 仅记录修改意见，我手动修改：不调用模型，状态会变为需修改。

### 图片生成中心

图片生成中心用于管理文章图片计划。

默认不接真实图片生成模型 API。如果在 `.env` 中配置 SiliconFlow，也可以直接在页面里点击“生成图片”。

基础功能支持：

- 顶部总览本篇图片计划数量、已生成数量、已有链接数量和已插入正文数量。
- 查看封面图提示词，封面图默认展开。
- 查看正文配图卡片，每张图展示对应小节、画面目标、比例、风格、状态、图注、提示词和负向提示词。
- 查看备用图，备用图默认折叠。
- 重新生成整篇图片计划。
- 只重写封面图或某张正文配图的提示词，已有图片不会被删除。
- 新增一张正文配图。
- 一键生成全部未生成图片，生成前需要确认会消耗图片额度。
- 一键应用已生成正文配图到正文，系统会按插入位置写入 `{{image:inline_x}}` 占位符。
- 复制图片提示词到外部图片工具。
- 手动上传图片，保存到本地。
- 配置云端图片存储后，一键生成稳定 HTTPS 发布链接。
- 手动填写稳定公网图片链接。
- 复制图片链接。
- 编辑图片说明、图注、画幅比例、图片风格和插入位置。
- 将正文配图插入正文，系统会写入 `{{image:inline_1}}` 这类占位符。
- 取消插入正文，不删除图片本身。
- 根据图片链接生成图文版 Markdown。

注意：

- 本地图片路径只用于本地预览和下载。
- 图文版 Markdown 不会把 `/Users/...` 或 `outputs/images/...` 这样的本地路径写入图片链接。
- 如果图片没有稳定 HTTPS 链接，图文版 Markdown 会保留“此处保留插图位置”的提示。
- SiliconFlow 返回的图片链接是临时签名链接，不能作为最终公众号发布链接。
- 如果希望复制到 Raphael 或公众号后台后稳定显示图片，请先生成稳定发布链接，或手动填写稳定公网图片链接。

### SiliconFlow 生图配置

如需使用 SiliconFlow 的 Z-Image-Turbo，在 `.env` 中配置：

```env
IMAGE_PROVIDER=siliconflow
IMAGE_MODEL=Tongyi-MAI/Z-Image-Turbo
IMAGE_GENERATION_ENABLED=true

SILICONFLOW_API_KEY=你的 SiliconFlow API Key
SILICONFLOW_API_BASE=https://api.siliconflow.com/v1
SILICONFLOW_IMAGE_MODEL=Tongyi-MAI/Z-Image-Turbo
```

说明：

- SiliconFlow 返回的图片链接可能有时效，系统会同时把图片下载到本地用于预览和下载。
- 如果要让 Raphael 或公众号后台稳定显示图片，请配置稳定图片存储。图文版 Markdown 只会使用稳定 HTTPS 链接，不会使用 SiliconFlow 临时链接。

```env
# 方案一：Streamlit Cloud 内置 media 链接，适合快速公开演示
IMAGE_STORAGE_PROVIDER=streamlit_media
IMAGE_PUBLIC_BASE_URL=https://你的应用域名

# 方案二：Cloudinary 免费图床，适合更稳定的长期使用
IMAGE_STORAGE_PROVIDER=cloudinary
CLOUDINARY_CLOUD_NAME=你的 cloud name
CLOUDINARY_UPLOAD_PRESET=你的 unsigned upload preset
CLOUDINARY_FOLDER=wechat-ai-workbench
```

配置后，在“图片生成中心”点击“生成稳定发布链接”，系统会把本地图片上传并写回稳定 HTTPS 链接。

### 5. 排版发布页

医生审核通过后才能进入。

页面提供：

- 文字版 Markdown：不包含图片。
- 图文版 Markdown：会把正文中的图片占位符替换成图片链接。
- 如果图片没有可访问链接，会保留插图位置提示。
- 复制 Markdown 按钮
- 下载 Markdown 文件
- 打开 Raphael 按钮
- 右侧 Raphael iframe 尝试内嵌
- 标记已发布

Raphael 地址：

```text
https://publish.raphael.app
```

如果右侧无法显示，请点击“打开 Raphael”新窗口使用。

如果右侧 iframe 中选择款式、设备预览按钮没有反应，也属于第三方 iframe 限制。完整操作建议使用“打开 Raphael（新窗口，推荐）”。

### 6. 知识库

知识库首页只展示知识库卡片，不展示长长的内容列表。

每张卡片展示：

- 知识库名称
- 已收录内容数量
- 最近更新时间
- 适合主题
- 基于它生成文章
- 管理

进入某个知识库详情后，可以查看“已收录内容”，并支持：

- 新增今日素材
- 上传 `.txt` 或 `.md` 资料
- 编辑知识库信息
- 查看、编辑、删除已收录内容

注意：不要上传原始咨询记录、真实个案故事或身份信息。

### 7. 新增今日素材

入口在：

- 内容工作台顶部
- 知识库页面
- 知识库详情页

使用方式：

1. 粘贴今天的新想法、观点、读者问题或脱敏后的常见现象。
2. 选择要放入的知识库。
3. 点击“AI 整理并加入知识库”。
4. 查看 AI 整理结果确认卡，包括标题、摘要、核心观点、标签、适合选题和风险等级。
5. 确认无误后点击“确认加入知识库”。

AI 整理标签时会优先使用已有标签，如果没有合适标签，最多新增 2 个核心新标签。

### 8. 基于知识库生成

这是当前核心生成流程。

页面分两步：

1. 选择知识库。
2. 选择生成方式。

支持三种方式：

- AI 自动推荐选题：系统根据知识库、历史稿件和历史选题推荐 5 个选题。
- 我自己输入主题：用户输入文章主题和目标读者，系统结合知识库直接生成文章。
- 选择知识库中的具体内容生成：用户勾选 1-5 条“已收录内容”，可以先推荐选题，也可以直接生成文章。

AI 推荐选题会写入 `topic_history`，用于后续避免重复。

选题去重时只传轻量历史信息：

- 历史文章标题
- 核心角度
- 状态
- 生成时间
- 历史推荐选题

不会把历史文章全文传给 DeepSeek。

生成出的文章会进入“已生成，待审核”状态，继续走现有编辑、审核、排版和 Raphael 流程。

### 9. 计划生成

用于维护“每周内容计划”，基于指定知识库手动生成本周选题。第一版只保存计划配置和立即生成，不做云端定时任务。

当前计划卡片会展示：

- 计划名称
- 本周主题方向
- 计划篇数
- 使用知识库
- 计划生成时间
- 启用 / 暂停状态
- 已生成文章数
- 待生成文章数

每个计划支持：

- 查看本周选题
- 立即生成本周稿件
- 暂停 / 启用计划
- 修改计划

新建计划时填写：

- 计划名称
- 使用哪个知识库
- 本周开始日期
- 本周主题方向
- 每周生成篇数，默认 3
- 重点标签
- 文章风格
- 文章长度，最长可选 4000 字
- 计划生成时间
- 生成后状态，默认“已生成，待审核”

点击计划卡片上的“立即生成本周稿件”后，系统会先调用 DeepSeek 生成选题，再批量生成文章任务，生成后的文章默认进入“已生成，待审核”。

点击“保存并立即生成本周选题”后，系统会只生成选题 JSON，并进入每周计划详情页。适合先看选题，再手动批量生成文章。

### 10. 每周计划详情页

展示本周计划和生成出的选题卡片。

每个选题包括：

- 选题标题
- 目标读者
- 核心观点
- 文章切入角度
- 风险标签
- 推荐理由

支持：

- 重新生成本周选题
- 单个选题生成文章任务
- 批量生成 3 篇文章任务
- 打开已生成的文章任务

从周计划生成出来的文章任务，会进入历史稿件和文章详情页，状态默认为“已生成，待审核”。后续仍然必须经过审核，才能复制 Markdown、进入排版发布页或标记发布。

## 任务状态

- 待输入
- 已生成
- 已生成，待审核
- 待修改
- 待审核
- 审核通过
- 排版中
- 已发布

## 数据保存在哪里

任务保存在本地 SQLite：

```text
data/tasks.db
```

如果部署时设置了 `DATABASE_URL`，任务会保存到外部 Postgres，不再依赖本地 SQLite。

知识库、稿件、选题历史和每周计划也保存在同一个数据库中：

- `knowledge_bases`：知识库
- `knowledge_items`：知识库里的已收录内容
- `article_drafts`：V2 稿件表
- `generated_images`：文章图片计划、图片链接、本地图片和图注
- `topic_history`：历史选题，用于后续去重
- `revision_history`：修改历史
- `tags`：标签库
- `app_settings`：基础设置
- `weekly_plans`：每周内容计划
- `weekly_topics`：每周计划生成的选题

正文会保存两个版本：

- `original_markdown`：DeepSeek 初次生成或人工编辑后的原始正文。
- `polished_markdown`：经过“一键优化排版”后的正文，复制到 Raphael 时优先使用。

Markdown 交付包保存在：

```text
outputs/{task_id}_完整交付包.md
```

如果数据库为空，系统会从 `input/tasks.csv` 导入一次旧任务。之后主要使用 SQLite。

## 演示流程

### 单篇文章演示

1. 打开“内容工作台”，点击“直接生成一篇”。
2. 填写一个脱敏科普卡。
3. 到“任务详情 / 编辑页”，点击“生成文案”。
4. 查看标题、摘要、正文 Markdown、图片提示词和风险审核报告。
5. 点击“一键美化排版”，生成更适合 Raphael 的美化 Markdown。
6. 如需调整，输入修改意见，点击“根据修改意见优化全文”。
7. 确认内容安全后，点击“提交医生审核”。
8. 点击“审核通过，进入排版”。
9. 如需配图，在“图片生成中心”生成图片。
10. 配置稳定图片存储后，点击“生成稳定发布链接”；未配置时，手动填写稳定 HTTPS 图片链接。
11. 确认图片卡片显示“稳定链接”。
12. 到“排版发布页”，选择“图文版 Markdown”并复制。
13. 点击“打开 Raphael”，粘贴 Markdown 做公众号排版预览。
14. 在公众号后台手动完成发布后，回到系统标记“已发布”。

### 图文版 Markdown 演示

1. 打开一篇已生成文章。
2. 查看右侧“图片生成中心”。
3. 复制封面图或正文配图提示词。
4. 到外部图片工具生成图片。
5. 回到系统，上传本地图片用于预览。
6. 如果想让 Raphael 或公众号后台直接显示图片，需要生成或填写稳定 HTTPS 图片链接。
7. 点击“复制图文版 Markdown”。
8. 打开 Raphael 粘贴预览。

### 知识库 + 每周 3 篇演示

1. 打开“内容工作台”。
2. 点击“新增今日素材”，粘贴一段脱敏内容。
3. 点击“AI 整理并加入知识库”。
4. 检查标题、摘要、核心观点、标签、适合选题和风险等级。
5. 点击“确认加入知识库”。
6. 回到“知识库”，进入知识库详情，确认内容已经出现在“已收录内容”里。
7. 点击“基于它生成文章”，进入“基于知识库生成”。
8. 选择“AI 自动推荐选题”，生成 5 个选题卡片。
9. 选择一个选题点击“生成这篇”，或点击“批量生成 3 篇”。
10. 生成出的文章会进入“已生成，待审核”，后续继续编辑、审核、一键优化排版和复制到 Raphael。

### 计划生成演示

1. 打开“计划生成”。
2. 创建一个“每周公众号内容计划”，选择知识库、主题方向、每周 3 篇、文章风格和文章长度。
3. 点击“保存并立即生成本周选题”，先查看 3 个选题卡片。
4. 在“每周计划详情页”查看 3 个选题卡片。
5. 点击“批量生成文章任务”。
6. 到“历史稿件”筛选“待审核”，继续编辑、审核、一键优化排版和复制到 Raphael。

如果已经有计划，也可以在“计划生成”页面直接点击“立即生成本周稿件”，系统会自动完成“生成选题 + 批量生成文章任务”。

一句话原则：

AI 负责辅助生成和优化，医生负责专业审核，Raphael 负责排版预览，最终发布必须由人工完成。
